6th APAC HPC-AI Competition 2023:
MPAS Atmosphere & BLOOM AI

Project Overview

This project was part of the prestigious 6th APAC HPC-AI Competition 2023, which involving two complex computational tasks run on the Gadi supercomputer. In the High-Performance Computing (HPC) track, the benchmarking and optimization of the MPAS Atmosphere climate model has been done by analyzing how simulation performance scales across increasing CPU nodes. Meanwhile, in the Artificial Intelligence (AI) track, a resource analysis for the BLOOM large language model has been conducted, which focusing on optimizing inference parameters like batch size to improve throughput and reduce latency.

This project's execution required deep analysis of the trade-offs between computational resources such as nodes, CPUs and memory, execution time and cost which often known as Service Units. It demonstrated a strong ability to manage and optimize large-scale parallel workloads in both scientific HPC computing and AI inference environments, and delivering key insights into efficient resource allocation for cutting-edge models.

Key Features

Dual-Expertise Showcase

Demonstrated proficiency in two distinct domains of modern computing: large-scale scientific HPC simulation and high-throughput AI inference optimization.

MPAS Atmosphere Scaling Analysis

Executed a systematic performance benchmark across 8, 16 and 32 nodes (up to 1,536 CPUs) while quantifying the relationship between node count, simulation speed, walltime and memory usage.

BLOOM 176B Inference Optimization

Developed and ran multiple experiments to analyze how varying batch sizes (4, 8, 12 and 16) directly impact GPU memory utilization, tokenization time and overall inference throughput.

Challenges & Solutions

Identified Challenges

Avoiding resource wastage on a shared supercomputer while determining the minimum viable resources to run the model without job failures.

Achieving optimal strong scaling efficiency without hitting resource contention or communication overheads between nodes.

Overcoming the steep learning curve of deploying a 176B parameter model on a unfamiliar HPC system with limited allocated resources.

Improving inference performance by reducing time per token without changing the model architecture or being granted more hardware resources.

Solutions & Action Plan

Implemented a cautious and iterative approach via PBS scripts, by starting with higher memory requests based on documentation to avoid errors, then refining based on results.

Conducted a scaling study with a scientifically significant run duration for each node count to prove that increased nodes reduced walltime and increase simulation speed despite higher total SU cost.

Rapidly acquired knowledge of the BLOOM architecture and Gadi's GPU environment through documentation and iterative testing by focusing on parameter tuning within the given constraints.

Optimized the batch size parameter to better utilize available GPU memory, by identifying that increasing the batch size from 8 to 16 led to a measurable decrease in tokenization time and improved throughput.
