Worries Away

Project Overview

DoClean! is a full-stack Flutter mobile application designed to bridge the gap between customers and housekeeping service providers. The app offers an intuitive platform for users to book personalized cleaning services effortlessly, while providing cleaning companies a channel to promote their services as well as reach a broader audience. This mobile app development implemented the Flutter framework and integrated Firebase to securely manage user data, booking orders and authentication processes, which then delivering a seamless and reliable user experience.

The mobile app was built on a viable business model by targeting working families, corporations and newly married couples. Its value proposition are to offer customers convenience and discounts while providing cleaning companies with increased visibility and customer acquisition. Thus, it will create a sustainable ecosystem facilitated by the app.

Key Features

One-Tap Booking System

Implemented an effortless booking interface, which allowing users to schedule personalized cleaning services with minimal steps.

Firebase-Backed Data Management

Securely stored and managed all customer personal details and booking orders in real-time using Firebase, to ensure data reliability and seamless synchronization.

Dual-Sided Marketplace Platform

Created a value proposition for both end-users such as discount vouchers and convenience, as well as service providers by increasing customer reach and promotional channels, as outlined in the Business Model Canvas.

Challenges & Solutions

Identified Challenges

Complex Android deployment process due to machine-specific Gradle and dependency conflicts, which often caused build failures across different development environments.

Designing a flexible data schema in Firebase to efficiently handle the relationship between users, bookings and service providers.

Translating the theoretical Business Model Canvas (BMC) into an actual solution app features that delivered value for both customer segments.

Solutions & Action Plan

Standardized the development environment using version-controlled Android Gradle configuration and clearly documented the setup process to ensure consistent and successful builds across all team members' machines.

Modeled Firestore collections to separate user data, booking orders and company profiles, which allowing scalable queries and secure access rules for different user types.

Mapped BMC components directly to app functionality such as implementing a voucher system for better value proposition.
