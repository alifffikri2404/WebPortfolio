MICTH Internal System

Project Overview

MICTH Internal System is a comprehensive and integrated suite of web applications developed to digitize and streamline the internal operations of MICTH. This project involved the integration of 4 separate systems including asset system, letter system, outstation system and booking system into a unified Single Sign-On (SSO) platform. The system encompassed full-stack development, which enhancing UI/UX with modern Bootstrap components, refactoring deprecated backend logic in PHP and integrating complex MySQL databases to create a cohesive and efficient internal ecosystem for managing assets, letters, outstation and bookings.

This internal system also required diligent maintenance and feature development across the entire software lifecycle. Key contributions included resolving critical vulnerabilities to ensure system stability during a physical server migration, optimizing Webmin for the virtual server hosting the domain i.micth.com, and adapting to evolving stakeholder requirements to deliver a robust, secure and user-friendly system that serves as the operational backbone for the entire MICTH organization.

Key Features

Unified Single Sign-On (SSO) Portal

Successfully integrated four distinct legacy systems: Asset, Letter, Outstation and Booking into one central platform that eliminating redundant logins and streamlining user access.

Comprehensive Administrative Dashboard

Provided admins with granular control over user privileges, asset lifecycle management from registration to disposal, and approval workflows for all booking and request systems.

Modernized & Responsive UI/UX

Revamped the outdated interface with a modern Bootstrap framework which significantly improving usability and cross-device compatibility for all employees.

Challenges & Solutions

Identified Challenges

Managing four separate databases with redundant data such as duplicate employee tables, which led to inefficient queries and maintenance headaches.

Upgrading a legacy codebase from deprecated PHP 5.5 to modern PHP 7.0+, which risked breaking existing functionality.

Ensuring zero data loss and minimal downtime during a complex physical server migration that involved reconciling different software dependencies including both Apache and MariaDB.

Adapting to frequently changing stakeholder requirements and shifting priorities based on new organizational needs.

Solutions & Action Plan

Designed and implemented a database integration plan to align and consolidate crucial tables while refactoring MySQL queries into a unified set of functions within the main application logic.

Conducted a systematic audit of the codebase, refactoring deprecated functions and ensuring compatibility with the newer PHP version without disrupting live services.

Meticulously planned the migration by utilizing Webmin to replicate configurations and performed rigorous testing in a staging environment, to mitigate risks and solve configuration conflicts proactively.

Adopted an agile and flexible development approach, maintaining clear communication with stakeholders to prioritize tasks and implement changes incrementally without compromising system stability.
